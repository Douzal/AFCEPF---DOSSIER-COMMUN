                                                                                                            here's my retry timer logic against dynamo... It's a bit more of a backoff, and limits to 10 tries.

                                                                                                            function getBackoffTime(num) {
                                                                                                            if (num < 3) return 250; // shorter wait for first time
                                                                                                            return 2 ** (num + 2) * 25 + 200; // 1000, 1800, 3400, 6600, 13000, 25800, 51400, 102600
                                                                                                            }

                                                                                                            num 1 is the first try, num 2 is before the first retry, forcing that to 250ms, after that it gets longer... this keeps it above a second to reduce write contention... It's generally better to muliply by a base, so that you get longer waits... similar for password retries.
